# Albero API

Small Express API for connecting the Albero website to the inventory SQL Server.

## Setup

1. Install dependencies:
   `npm install`
2. Review `.env`
3. Start the server:
   `npm start`

The API listens on `http://0.0.0.0:5000` by default so you can test it over Tailscale during development.

## Available endpoints

- `GET /health`
- `GET /health/db`
- `GET /api/products`
- `GET /api/products/:productId`
- `GET /api/catalog/facets`
- `POST /api/stock/lookup`
- `GET /api/meta/tables`
- `GET /api/meta/tables/:schema/:table/columns`
- `GET /api/meta/tables/:schema/:table/rows?limit=20`

The `/api/meta/*` routes are intended for development only so you can inspect the database schema and sample rows while we figure out which inventory tables the website actually needs.

## Catalog API

### `GET /api/products`

Returns paginated storefront products grouped by `MD_ID`.

Supported query params:

- `gender=uomo|donna`
- `category=abbigliamento|accessori|calzature`
- `brand=LIU JO SHOES`
- `seasonCode=261`
- `availability=all|available|unavailable`
- `q=<search text>`
- `ids=mdl_123,mdl_456`
- `page=1`
- `pageSize=24`
- `sort=season-desc|season-asc|name-asc|name-desc|price-asc|price-desc`

### `GET /api/products/:productId`

Returns a single grouped product with:

- cleaned `descriptionShort`
- cleaned `featureLines`
- aggregated `images`
- color variants with exact per-size stock

### `POST /api/stock/lookup`

Request body:

```json
{
  "skus": ["7G5015PX33101111TUMBL"]
}
```

Returns live exact size stock derived from net `ESI_ETICHETTE` quantities in `BARCODE_ESISTENZA_RFID`.

### `GET /api/catalog/facets`

Returns brand, category, subtype, season, and availability counts for filters.

## Image Sync

The API can now resolve image URLs from a local manifest generated by a sync job.

### What the sync does

- reads active image paths from `Articoli_Su_Sito_Plus`
- rewrites the SQL UNC root to the server's real local photo path when needed
- uploads each reachable file to Vercel Blob
- writes a manifest at `IMAGE_SYNC_MANIFEST_PATH`
- lets the API return Blob URLs through the existing product image fields

### Required environment variables for sync

- `BLOB_STORE_ID`
- `BLOB_READ_WRITE_TOKEN`
  - or `BLOB_PUBLIC_READ_WRITE_TOKEN` if you want to keep your existing naming
- `IMAGE_SOURCE_DB_ROOT`
  - the root prefix stored in SQL, usually `\\SERVER-ATELIER\Foto`
- `IMAGE_SOURCE_LOCAL_ROOT`
  - the actual path on the machine running the sync
- optional `IMAGE_SYNC_BLOB_PREFIX`
- optional `IMAGE_SYNC_MANIFEST_PATH`

### Example server setup

If SQL stores:

```text
\\SERVER-ATELIER\Foto\261\ax armani exchange\file.jpg
```

and the server running the sync can see that same folder locally as:

```text
D:\Foto
```

set:

```env
IMAGE_SOURCE_DB_ROOT=\\SERVER-ATELIER\Foto
IMAGE_SOURCE_LOCAL_ROOT=D:\Foto
```

### Commands

- Dry run:
  `npm run sync-images -- --dry-run --limit=20`
- Real sync:
  `npm run sync-images`
- Force re-upload:
  `npm run sync-images -- --force`

## Stock Cache

For production, install the SQL-side stock cache and refresh it on a schedule:

1. Run `npm run install-stock-cache` with a SQL login that can create tables/procedures in `dbo`.
2. If the installer is not run as the API login, set `STOCK_CACHE_GRANT_USER=<api_db_user>` before installing so the API receives `SELECT` and `EXECUTE` permissions.
3. Run `npm run refresh-stock-cache` after installation, then schedule it with Windows Task Scheduler or SQL Agent.

When `dbo.Albero_Stock_Cache` exists and has rows, the API reads stock from that table. Until then it falls back to the live RFID aggregation. Set `REQUIRE_SQL_STOCK_CACHE=true` in production if a missing cache should fail loudly.

## Runtime Discount Data

Discount rules and coupon redemptions can be stored in the same SQL Server used by this API.

1. Set the same `ALBERO_API_RUNTIME_SECRET` value on `albero-api` and `albero-site`.
2. Run `npm run install-runtime-discounts` to create `dbo.Albero_Discount_Rules`, `dbo.Albero_Coupon_Redemptions`, and `dbo.Albero_Admin_Audit_Log`.
3. Run `npm run migrate-runtime-discounts` once to import the existing Blob JSON discount data. Use `-- --force` only when intentionally replacing existing DB rules.
4. Keep Blob for images/files; runtime discount reads and writes should go through the API routes under `/api/runtime/discounts/*`.

## Notes

- Product ids use `mdl_<MD_ID>`.
- Variant ids use `var_<VA_ID>`.
- Variant `sku` is built as `trim(MD_CODICE) + trim(VA_CODICE)`.
- Catalog metadata comes from `dbo.Articoli_Su_Sito_Plus`; sellable stock comes from net RFID existence, not raw barcode-label rows.
- Unavailable products are visible only for the current season. Older-season products with zero stock are pruned from catalog totals, facets, lists, and details.
- If the image manifest exists, product image `url` values come from it.
- If the manifest does not exist yet, image `url` stays `null` and `sourcePath` is still returned in detailed image objects.
